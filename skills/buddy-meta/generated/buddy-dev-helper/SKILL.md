# Buddy Dev Helper Skill

## Опис
Розробницький помічник для аналізу проєктів та отримання рекомендованих структур.

## Дії (actions)

### 1. scaffold
Повертає рекомендовану структуру проєкту для обраного типу.

**Аргументи:**
- `type` — тип проєкту: `nextjs`, `python`, `node`

**Приклад:**
```
buddy-dev-helper scaffold nextjs
```

**Відповідь:**
```json
{
  "project_type": "Next.js",
  "structure": [".next/", "app/", "components/", ...]
}
```

---

### 2. deps
Читає залежності з package.json або requirements.txt.

**Аргументи:**
- `path` — шлях до проєкту

**Приклад:**
```
buddy-dev-helper deps /home/user/my-project
```

**Відповідь:**
```json
{
  "project_type": "Node.js",
  "file": "package.json",
  "dependencies": ["react", "next"],
  "devDependencies": ["typescript", "eslint"],
  "total_count": 4
}
```

---

### 3. git-status
Перевіряє чи є директорія Git-репозиторієм.

**Аргументи:**
- `path` — шлях до репозиторію

**Приклад:**
```
buddy-dev-helper git-status /home/user/my-project
```

**Відповідь:**
```json
{
  "is_git_repo": true,
  "repo_name": "my-project",
  "branch": "main",
  "remotes": ["https://github.com/user/repo.git"]
}
```

## Помилки
- `Невідомий тип проєкту` — для scaffold вказано невірний тип
- `Шлях не існує` — для deps/git-status вказано невірний шлях
- `Файли залежностей не знайдено` — у проєкті немає package.json або requirements.txt
